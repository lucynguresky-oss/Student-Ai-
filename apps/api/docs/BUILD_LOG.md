## Schema additions beyond the original brief (all justified)

| Addition | Why |
|---|---|
| `PlacementQuestion` model | The brief's placement quiz (§5.6) needs a question bank with correct-answer indices kept server-side. Seeded with 65 questions across 13 tracks. |
| `UserStreak` model | Gamification defaults (§10) require streak state (current/longest/last-activity) initialized at 0 on onboarding completion. |
| `Session.rotatedAt` | Bookkeeping for refresh rotation (§5.1). |
| `Session.prevRefreshTokenHash` (unique) | Enables **refresh reuse-detection**: a replay of the immediately-previous token is caught and revokes the whole family. Without it, overwritten hashes couldn't be traced to their family. |

OTP codes are intentionally **not** modelled — they live in Redis only (§4).

## GLOBALIZATION (mid-build pivot — "the app is for everyone, the whole globe")

Original brief was Kenya-first. Retrofitted to a worldwide audience:

- **Phone:** replaced the Kenya-only regex with **libphonenumber-js**. `normalizePhone(raw,
  country?)` → `{ok, e164, country, isMobile}` for all 252 territories. `normalizeKenyanPhone`
  retained only as a backward-compat wrapper.
- **Geo/language data:** embedded a CLDR-derived dataset (`packages/validation/src/geo.data.ts`,
  generated by `scripts/gen-geo.cjs` from `countries-list`): **252 countries** (ISO code, name,
  native, calling code, flag, continent, spoken languages) + **185 languages** (ISO code, English +
  native name, RTL flag).
- **Validation:** `country`/`language`/`contentLanguage` now validate against the full ISO lists
  (was `en`/`sw` enum). Any language is a valid *preference*; UI falls back to English where no
  translation bundle exists (`SHIPPED_UI_LANGUAGES`).
- **Reference endpoints:** `GET /reference/countries`, `/languages`, `/country-languages` feed the
  client's global pickers.
- **Track catalogue:** broadened to world languages (English, Spanish, French, Kiswahili, Mandarin,
  Arabic) + universal academics + skills; placement banks added for each new language track.
- **Timezone:** client-detected (IANA), not hardcoded to `Africa/Nairobi`.
- **Copy:** region-neutral (no hardcoded national exam names in product copy).
- **RTL:** languages carry an `rtl` flag; the web app must mirror layout for Arabic/Hebrew/etc.
